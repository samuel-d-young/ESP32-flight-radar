Setup — three steps

01

Plug in and install

Connect your board over USB, pick the matching card above, and click Install. After flashing the browser will automatically show a Configure WiFi dialog — enter your home network name and password right there, no separate setup step needed. The installer detects ESP8266, ESP32, and ESP32-S3 automatically and flashes the right build.

02

Set your location

After connecting to WiFi the device's IP address shows on the display for a few seconds. Visit that IP in any browser on the same network to set your home latitude, longitude, and radar range. You can also change WiFi here any time — no reflash needed.

03

Change settings any time

Once connected, the device runs a small web page at its local IP address. Open a browser on the same network, visit that IP, and you can update your home location or radar range without reflashing. The IP is shown on the display for a few seconds on every boot. To change the Wi-Fi network itself, hold the board's BOOT button for 5 seconds to re-open the setup portal.

04

Wire your display (if it needs wiring)

The round and rectangular builds need the display wired up — pick your board and exact display in the display library below. The ESP32-4848S040 is a self-contained module, so this step doesn't apply to it — just power and go.

Display library

Four displays have real, compiled firmware now — GC9A01, ILI9341, the ESP32-4848S040, and the Eosnow 1.85″ round. Everything else below is wiring reference for a future custom build, not something the install buttons above can flash.

Pick your board, then your display — GPIO numbers differ between ESP8266 NodeMCU and ESP32 dev boards even where pin labels look similar (the ESP32-4848S040 fixes this automatically since it's a single all-in-one module).

Not supported, and can't be: classic AVR Arduinos (Uno, Nano, Mega, Leonardo) don't have the Espressif serial bootloader that browser-based flashing depends on — no browser tool can flash them, regardless of how this page is built. They'd need a completely different, AVR-specific firmware and a desktop flashing tool (e.g. avrdude) instead.

Built with ESP Web Tools, flashing over Web Serial — nothing here leaves your machine except the ESP8266 itself, which talks only to adsb.lol once it's online.
